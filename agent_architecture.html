— Intent Classification —
🌐 General Info
Direct conversational queries — no external data needed
Prompt sent directly to LLM with system context
Single LLM call, no tool use
e.g. "What is PUE?" / "How does cooling work?"
📚 Knowledge (RAG)
Company-specific queries via PDF vector store
Query embedded via nomic-embed-text
FAISS semantic search → top-K chunks
Chunks injected into LLM context
e.g. "ACO v3.7 cooling thresholds?"
⚙️ Action (Zabbix)
Live metric requests via Zabbix client
LLM extracts tool name + params from query
Allowlist: get_cpu_usage / get_power_usage
Raw metric → LLM for NL explanation
e.g. "Show PROD1 CPU" / "RACK-A1 power"

ReAct Agent Loop — POST /agent
🔁 ReAct Agent Loop
Plain Python · no framework · prompt formatting + JSON parsing
MAX_STEPS = 6
💭 Thought
LLM reasons what to do next. Emits FORMAT A JSON to call a tool.
→
🔧 Action
@add_tool registry dispatches the named function with extracted params.
→
👁 Observation
Result injected back as user turn. LLM picks: another tool (A) or final answer (B).
↩
Observation appended to scratchpad (message history) → loop repeats · duplicate (action, input) pairs abort · FORMAT B emits final_answer and terminates · response shape: {final_answer, steps[], total_steps}
Registered tools — stubs → swap bodies for real Zabbix calls

Session Persistence — Redis
🗃 Redis Session Store
async redis.asyncio · every /chat turn persisted · sidebar + history served from Redis
Redis
chat:{user_id}:{session_id}
Type: List (RPUSH / LRANGE)
Ordered MessageStore objects — role, text, timestamp (epoch). Full turn history per session.
chats:{user_id}
Type: Hash (HSET / HGETALL)
ChatMetadata per session — session_id, title (first 30 chars), updated_at. Drives sidebar sort.
GET /sidebar → HGETALL + sort by updated_at GET /chat-history/{id} → LRANGE 0 -1 POST /chat → RPUSH + HSET upsert
