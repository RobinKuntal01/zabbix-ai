— Intent Classification —
🌐 General Info
Direct conversational queries — no external data needed
Prompt sent directly to LLM with system context
Single LLM call, no tool use
e.g. "What is a data center?" / "How does PUE work?"
📚 Knowledge (RAG)
Company-specific queries resolved via PDF knowledge base
Query embedded via nomic-embed-text
Semantic search on FAISS vector index
Top-K chunks injected into LLM context
e.g. "What's our SLA for Tier-3 racks?"
⚙️ Action (Tool Call)
Operational requests that fetch live Zabbix metrics
LLM extracts tool name + params from query
Dispatches: get_cpu_usage / get_power_usage
Raw metric → LLM for natural language explanation
e.g. "Show PROD1 CPU" / "RACK-A1 power draw"

🔁 ReAct Agent Loop
Multi-step reasoning · scratchpad-as-message-history · loop detection
MAX_STEPS = 6
💭 Thought
LLM reasons about what to do next. Emits FORMAT A JSON to call a tool.
→
🔧 Action
Tool dispatcher calls registered function with extracted params.
→
👁 Observation
Tool result injected back as user turn. LLM decides: next tool (A) or answer (B).
↩
Observation appended to scratchpad (message history) → loop repeats · duplicate call detection prevents infinite loops · FORMAT B emits final_answer and terminates
Registered tools — stub stubs → swap to real Zabbix calls
get_server_cpu get_rack_power list_servers_in_rack list_available_racks get_cooling_status
